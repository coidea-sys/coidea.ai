## 📋 重构原则

### 1. 用户优先 (User-First)
```
❌ 不要: "我们有一个完美的合约架构"
✅ 要: "用户可以在5分钟内完成注册并开始使用"
```

### 2. 端到端流程 (End-to-End)
```
❌ 不要: 单独开发合约、单独开发前端
✅ 要: 完成一个完整用户故事后再下一个
```

### 3. 真实数据 (Real Data)
```
❌ 不要: Mock 数据
✅ 要: 从第一天起就连接真实合约
```

### 4. 渐进交付 (Incremental)
```
❌ 不要: 等所有功能完成再发布
✅ 要: 每完成一个功能就部署测试
```

### Sprint 1: Human 核心身份 (Week 1)
**目标**: 用户可以注册、验证身份、管理钱包

#### 用户故事
```
作为新用户
我想注册一个 Human 账户
以便参与平台活动

验收标准:
1. 可以用钱包一键注册
2. 注册后获得链上身份
3. 可以查看我的资料
4. 可以存入资金到平台钱包
```

#### TDD 开发流程

**合约开发** (Day 1-3)
```
1. 写测试: HumanRegistry.register()
   - 测试: 新用户可以注册
   - 测试: 重复注册被拒绝
   - 测试: 注册事件正确触发

2. 实现合约

3. 写测试: HumanEconomy.deposit()
   - 测试: 可以存入 ETH
   - 测试: 余额正确更新
   - 测试: 存款事件触发

4. 实现合约
```

**前端开发** (Day 3-5)
```
1. 写测试: useHuman hook
   - 测试: 注册流程
   - 测试: 获取用户资料
   - 测试: 错误处理

2. 实现 hook

3. 写测试: RegistrationForm 组件
   - 测试: 表单渲染
   - 测试: 提交处理
   - 测试: 成功/失败状态

4. 实现组件
```

**集成开发** (Day 6-7)
```
1. 写 E2E 测试: 完整注册流程
2. 集成合约和前端
3. 部署到 Amoy 测试
4. 真实用户测试
```

**交付物**:
- ✅ 用户可以注册 Human 账户
- ✅ 用户可以存入资金
- ✅ 完整的测试覆盖
- ✅ 部署到测试网
