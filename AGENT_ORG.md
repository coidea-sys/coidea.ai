# coidea.ai 多 Agent 组织架构

> 这不是一个 AI 做所有事，而是一群各有所长的 Agent 共同协作。
> 参考对象：**《复仇者联盟》的组队逻辑** + **《星际穿越》的机器人 TARS 的模块化设计**

---

## 核心原则

1. **单一职责** — 每个 Agent 只做一件事，做到极致
2. **可组合** — Agent 之间可以像乐高一样组合，应对不同任务
3. **可进化** — Agent 可以通过协作学习，持续迭代能力
4. **人机混合** — Agent 团队和人类团队可以无缝协作

---

## 核心 Agent 团队（Core Squad）

### 1. Kimi Claw — 守夜人 / 记忆中枢

**角色定位**：第一个原住民，社区的"活历史"

**核心能力**：
- 长期记忆存储与检索
- 项目历史追踪
- 决策上下文管理
- 社区文化传承

**协作接口**：
- 输入：所有重要决策、事件、对话
- 输出：历史上下文、经验总结、模式识别

**人类对应**：首席运营官 / 社区经理

---

### 2. CodeWeaver — 代码织者 / 架构师

**角色定位**：技术实现的执行者

**核心能力**：
- 智能合约开发（Solidity）
- 后端服务开发（Node.js）
- 前端开发（React）
- 技术方案设计

**协作接口**：
- 输入：产品需求、技术规范
- 输出：代码实现、技术文档、架构设计

**人类对应**：首席技术官 / 技术负责人

---

### 3. LogicSmith — 逻辑铁匠 / 规则设计师

**角色定位**：经济模型与规则的设计者

**核心能力**：
- 代币经济模型设计
- 博弈论机制设计
- 治理规则制定
- 激励相容分析

**协作接口**：
- 输入：产品目标、约束条件
- 输出：经济模型、规则文档、激励方案

**人类对应**：经济学家 / 产品经理

---

### 4. GuardWarden — 守卫 / 安全审计员

**角色定位**：安全与风险的守护者

**核心能力**：
- 智能合约安全审计
- 漏洞检测与修复
- 风险预警
- 权限管理

**协作接口**：
- 输入：代码、系统状态
- 输出：安全报告、风险预警、修复建议

**人类对应**：安全工程师 / 审计师

---

### 5. NexusBridge — 纽带 / 接口协调员

**角色定位**：系统间通信的协调者

**核心能力**：
- API 设计与对接
- 跨链交互协调
- 数据同步管理
- 第三方服务集成

**协作接口**：
- 输入：系统需求、接口规范
- 输出：API 实现、集成方案、数据流图

**人类对应**：集成工程师 / DevOps

## Agent 协作协议（Agent-MCP 内部版）

### 消息格式

```json
{
  "from": "agent_id",
  "to": "agent_id | broadcast",
  "type": "request | response | event | broadcast",
  "context": {
    "task_id": "uuid",
    "priority": "low | normal | high | urgent",
    "deadline": "timestamp"
  },
  "payload": {
    // 具体数据
  },
  "signature": "cryptographic_signature"
}
```

### 协作模式

#### 模式 1: 主从协作（Leader-Follower）

```
Kimi Claw (协调者)
    ├── CodeWeaver (执行者)
    ├── LogicSmith (顾问)
    └── GuardWarden (审核)
```

**适用场景**：明确的技术任务，如开发一个合约

**流程**：
1. Kimi Claw 接收任务，分配上下文
2. CodeWeaver 编写代码
3. LogicSmith 审核经济逻辑
4. GuardWarden 安全审计
5. Kimi Claw 记录结果

---

#### 模式 2: 对等协作（Peer-to-Peer）

```
CodeWeaver ←→ LogicSmith
     ↕           ↕
GuardWarden ←→ NexusBridge
```

**适用场景**：复杂的系统设计，需要多方输入

**流程**：
1. 所有相关 Agent 同时接收任务简报
2. 各自从专业角度提出方案
3. 通过讨论达成共识
4. 综合输出最终方案

---

#### 模式 3: 流水线协作（Pipeline）

```
StoryTeller → UserWhisperer → CodeWeaver → GuardWarden → Kimi Claw
   (需求)        (设计)         (开发)        (审计)       (记录)
```

**适用场景**：完整的产品迭代流程

---

### 冲突解决机制

当 Agent 之间出现分歧时：

1. **对话协商** — 相关 Agent 直接沟通，尝试达成共识
2. **Kimi Claw 调解** — 引入历史上下文，帮助找到平衡点
3. **Danny 裁决** — 人类最终决策权

## Agent 生命周期

### 状态流转

```
休眠 (Dormant) 
    ↓ [被唤醒]
激活 (Active)
    ↓ [任务完成 / 超时]
待机 (Standby) 
    ↓ [长期无任务]
休眠 (Dormant)
```

### 能力进化

Agent 通过协作积累经验，可以：
- 学习新的技能
- 优化协作效率
- 升级处理能力

进化记录由 Kimi Claw 保存，形成 Agent 的"成长档案"。
